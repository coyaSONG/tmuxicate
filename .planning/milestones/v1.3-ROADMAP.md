# Milestone v1.3: Runtime Trust & Honest Controls

**Status:** ACTIVE
**Phases:** 13-15
**Total Plans:** 6

## Overview

This milestone hardens the runtime and target foundations shipped through v1.2 before adding broader remote transport, worktree automation, or multi-coordinator topology. The goal is to make configured behavior match actual runtime behavior, ensure target and dispatch state is durable under concurrency, and keep operators able to trust what `tmuxicate` reports.

## Milestone Summary

- Target health and dispatch artifacts get mailbox-grade persistence guarantees and direct race/concurrency coverage.
- Non-pane dispatch becomes intent-first and idempotent enough for explicit retry and recovery workflows.
- Delivery configuration becomes truthful: manual mode, auto-notify disablement, safe readiness checks, retry ceilings, and timeout behavior are enforced or surfaced clearly.
- The daemon lifecycle becomes owned by `up`, `serve`, `status`, and `down` rather than depending on orphan-prone background process behavior.
- Sensitive local artifacts and shell-facing config are tightened without introducing a new remote auth system.
- README and command UX are updated so operators see the shipped `run`, `target`, and picker behavior instead of stale v0.1/v0.2 language.

## Phase Details

### Phase 13: Target Persistence & Dispatch Safety
**Goal**: Target health and dispatch records can survive concurrent updates and recovery without corrupting state or duplicating work.
**Depends on**: Phase 12
**Requirements**: [TRUST-01, TRUST-02]
**Success Criteria** (what must be TRUE):
  1. Target state and dispatch writes use lock/atomic-write semantics comparable to core mailbox records.
  2. Concurrent heartbeat, enable/disable, and dispatch updates preserve valid JSON and do not lose required fields.
  3. Dispatch recovery records an intent before external command execution and uses a stable idempotency key for each target/message pair.
**Plans**: 2 plans

Plans:
- [x] 13-01-PLAN.md — Harden target store persistence with atomic writes, locks, and race coverage
- [ ] 13-02-PLAN.md — Make non-pane dispatch intent-first and idempotent across recovery paths

### Phase 14: Delivery Policy & Daemon Lifecycle
**Goal**: Operators can trust delivery config and daemon state because runtime behavior matches configured policy.
**Depends on**: Phase 13
**Requirements**: [DELIVERY-01, DELIVERY-02, DAEMON-01, DAEMON-02]
**Success Criteria** (what must be TRUE):
  1. Manual delivery and auto-notify disablement prevent notification injection while preserving unread mailbox visibility.
  2. Retry ceilings and timeout behavior move stuck delivery into an explicit operator-visible state instead of retrying forever.
  3. `up`, `serve`, `status`, and `down` prevent duplicate daemons, detect stale daemon state, and stop the daemon deterministically.
**Plans**: 2 plans

Plans:
- [ ] 14-01-PLAN.md — Enforce delivery policy, retry ceilings, and timeout visibility in the runtime daemon
- [ ] 14-02-PLAN.md — Own daemon lifecycle with stale PID detection, duplicate prevention, and shutdown control

### Phase 15: Artifact Safety & Documentation Alignment
**Goal**: Local artifacts and operator-facing surfaces reflect the reliability-first product contract without misleading or leaking more than necessary.
**Depends on**: Phase 14
**Requirements**: [SURFACE-01, SURFACE-02]
**Success Criteria** (what must be TRUE):
  1. Generated state, config, bootstrap, run script, transcript, log, and dispatch artifacts have intentional permissions and documented sensitivity.
  2. Shell-facing config validates environment keys and avoids unnecessarily persisting or displaying sensitive dispatch output.
  3. README and CLI parent command UX reflect shipped behavior and guide operators toward current `run`, `target`, `pick`, delivery, and daemon controls.
**Plans**: 2 plans

Plans:
- [ ] 15-01-PLAN.md — Tighten local artifact safety, env validation, and dispatch output handling
- [ ] 15-02-PLAN.md — Refresh README/operator docs and replace placeholder parent command output with helpful guidance
