# Phase 13 Plan 01: Target Store Durability

## Objective

Harden target state, heartbeat, and dispatch persistence so concurrent target operations preserve valid, inspectable JSON and do not lose required state.

## Tasks

1. Add focused tests for `internal/mailbox/target_store.go` covering default state, heartbeat logs, dispatch records, list ordering, validation errors, and concurrent target updates.
2. Replace plain target state and dispatch writes with lock-protected atomic writes that preserve human-readable JSON while avoiding truncated or lost files.
3. Add or reuse fsync/parent-directory durability helpers where practical without changing the public mailbox protocol.

## Verification

- `go test ./internal/mailbox -count=1 -race` passes.
- Target state and dispatch records remain valid JSON after concurrent heartbeat, state, and dispatch writes.
- Existing target/session tests continue to pass without local pane behavior changes.

## Guardrails

- Preserve existing target file paths and JSON schema unless a schema extension is required and tested.
- Do not add authenticated transport, worker bootstrap, or non-shell dispatch in this plan.
- Inspect pre-existing WIP in target files before editing and avoid overwriting unrelated changes.
