## Accepted Tech Debt

- Non-local execution uses operator-configured command dispatch plus shared session state, not an authenticated transport layer or worker bootstrap service.
- Remote lifecycle parity still assumes remote workers call the existing mailbox/task commands or emit the same durable state transitions; the system does not synthesize missing lifecycle events.
- The local daemon remains pane-scoped only; non-pane targets are observable through target state and dispatch artifacts rather than tmux-native health probes.
