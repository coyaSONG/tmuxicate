# Requirements: tmuxicate

**Defined:** 2026-04-11
**Core Value:** A human can coordinate multiple terminal agents through a reliable, observable workflow where the coordinator keeps work moving without hiding what happened.

## v1.2 Requirements

### Remote Dispatch

- [x] **REMOTE-01**: Coordinator can persist enough target transport configuration to distinguish concrete remote execution from local pane-backed execution without breaking existing local targets
- [x] **REMOTE-02**: Coordinator can dispatch eligible work through a concrete remote execution path while preserving canonical run, task, and routing artifacts

### Target Health

- [x] **HEALTH-01**: Operator can inspect durable readiness, heartbeat, and capability state for remote targets before routing work
- [x] **HEALTH-02**: Remote execution emits durable lifecycle state that keeps `run show`, summaries, and timeline projections consistent with local execution

### Operator Control

- [x] **CTRL-01**: Operator can explicitly disable, quarantine, or recover a remote target without mutating historical run artifacts
- [x] **CTRL-02**: Coordinator explains target selection, rejection, or reroute decisions when remote targets are unavailable or unhealthy

## Deferred Requirements

- **DEFER-01**: Coordinator can manage nested teams or multiple coordinators within one durable workflow graph
- **DEFER-02**: Operators can compare, rebalance, and inspect work across multiple coordinator runs or teams
- **DEFER-03**: Coordinator can evolve adaptive signals into richer inspectable recommendations or auto-tuning without hiding control boundaries

## Outcome Notes

- Remote execution shipped as an operator-configured command dispatch path layered onto the existing mailbox-backed run/task model.
- Target health shipped as durable target state, heartbeat timeout handling, and operator-visible status surfaces rather than daemon-managed remote probing.
- Operator control shipped as explicit target CLI commands plus reroute and redispatch behavior with durable excluded-target evidence.
- Remote lifecycle parity shipped by reusing the existing task lifecycle and `state.jsonl` contract instead of introducing a second remote event protocol.
