<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| completed run graph -> adaptive preference artifact | Historical task/review/blocker outcomes become reusable routing inputs |
| operator config -> adaptive scoring | Manual preference weights and owner choices can alter future ranking if not validated strictly |
| root completion hook -> durable cross-run state | One command path writes coordinator-wide preferences that later route decisions depend on |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | T | `internal/config/loader.go` | mitigate | Validate adaptive task classes, normalized domains, preferred owners, and non-negative weights before any preference artifact is written. |
| T-06-02 | T | `internal/session/run_adaptive.go` | mitigate | Build preferences only from canonical `LoadRunGraph` plus `BuildRunSummary` outputs and reject pending/incomplete items from scoring. |
| T-06-03 | R | adaptive preference artifact | mitigate | Persist run/task/message evidence refs on every non-zero historical signal so later route decisions can be audited back to source lineage. |
| T-06-04 | I | root completion hook | accept | Root-only refresh exposes coordinator-scoped preference data already reconstructible from run artifacts; no secrets or transcript content are added. |
| T-06-05 | E | manual preference inputs | mitigate | Require preferred owners to remain configured agents and keep the artifact outside message/receipt mutation paths so manual boosts cannot bypass allowed-owner boundaries. |
</threat_model>
