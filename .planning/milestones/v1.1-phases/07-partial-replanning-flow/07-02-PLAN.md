<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| blocker resolve CLI -> replacement task creation | Operator-supplied replan fields become a new child task inside an existing coordinator run. |
| partial replan artifact -> run-show/summary rebuild | Durable lineage controls which task is treated as superseded versus current replacement work. |
| superseded receipt -> replacement task | One work item is retired and another takes over the source-task lineage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | T | `internal/session/blocker_resolve.go` | mitigate | Allow `partial_replan` only from escalated blocker cases, require explicit replacement task fields, and reject a second replan artifact for the same source task. |
| T-07-02-02 | E | routing / owner selection | mitigate | Create replacement work through the existing routed child-task path so allowed-owner checks, duplicate guards, and deterministic owner selection remain authoritative. |
| T-07-02-03 | R | `internal/session/run_rebuild.go` | mitigate | Validate blocker-resolution and task/message links against the immutable replan artifact and return `coordinator artifact mismatch` instead of rendering partial lineage. |
| T-07-02-04 | D | superseded active receipt | mitigate | Move the superseded receipt out of `active` before replacement execution so the old task cannot continue silently in parallel with the replacement task. |
</threat_model>
