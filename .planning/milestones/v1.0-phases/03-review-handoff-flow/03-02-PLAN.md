<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reviewer CLI -> `internal/session/review_response.go` | untrusted `message_id`, `agent`, `outcome`, and response body enter the review-response workflow here |
| review-response workflow -> canonical handoff artifact | terminal review outcome is recorded here and must not be spoofed or overwritten by the wrong agent |
| rebuild/render -> mailbox and coordinator artifacts | operator-visible review chains are reconstructed here and must reject drift instead of presenting false lineage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-05 | T | `internal/session/review_response.go` and `internal/mailbox/coordinator_store.go` | mitigate | Update only the existing `reviews/<source-task-id>.yaml` artifact, reject `status=responded` replays, and never create a second handoff or overwrite the source task state when a reviewer responds. |
| T-03-06 | T | `internal/session/run_rebuild.go` | mitigate | Validate review-response `kind`, `reply_to`, thread, review-task owner, and task/message existence before rendering; fail with coordinator artifact mismatch on any broken review chain instead of showing partial output. |
