<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator goal -> coordinator run contract | Untrusted CLI input becomes durable run metadata |
| coordinator task spec -> child-task contract | Structured task fields become on-disk graph artifacts |
| generated IDs -> filesystem paths | Record identifiers are joined into state-dir paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `internal/protocol/coordinator.go` | mitigate | Validate every required field and generated ID prefix before any writer is added; reject blank or malformed run/task records. |
| T-01-02 | T | `internal/mailbox/paths.go` | mitigate | Build coordinator artifact paths from generated `run_` and `task_` IDs only; do not accept raw path fragments from CLI inputs. |
| T-01-03 | I | coordinator durable records | mitigate | Keep the canonical contract limited to goal/task metadata, references, and team snapshot fields; do not add env vars, transcript paths, or secret-bearing runtime details. |
| T-01-04 | R | contract drift across plans | mitigate | Pin the schema and path contract in `internal/session/run_test.go` so later plans cannot silently diverge from the canonical on-disk layout. |
</threat_model>
