## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk artifacts -> rebuild model | Canonical run/task YAML is joined with mailbox and state files |
| rebuild model -> operator CLI output | Aggregated state is rendered for humans and must not hide divergence |
| receipt/state files -> run tree summary | Mutable state is merged into durable run/task lineage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | R | `internal/session/run_rebuild.go` | mitigate | Rebuild only from disk artifacts and fail loudly on missing run/task/message/dependency joins instead of silently dropping broken nodes. |
| T-01-10 | T | `internal/session/run_rebuild.go` | mitigate | Validate dependency IDs and message references while loading task YAML so malformed artifacts cannot produce an incorrect run tree. |
| T-01-11 | I | `run show` output | mitigate | Render only durable IDs, ownership, goals, expected outputs, and compact state labels; never print transcript content, env values, or raw runtime logs. |
| T-01-12 | R | operator inspection path | mitigate | Display `Run`, `Task`, `Owner`, `State`, and `Message` labels from canonical data so the operator can audit lineage without transcript spelunking per D-07 and D-08. |
</threat_model>
