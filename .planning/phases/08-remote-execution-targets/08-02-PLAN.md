<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI -> placement preview/output | Preview output must match persisted routing decisions so operators do not act on stale or divergent placement information |
| local tmux runtime -> non-local execution targets | Session startup and notifications cross from pane-managed local agents to agents that intentionally have no local pane/runtime ownership |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | T | `cmd/tmuxicate/main.go`, `internal/session/run.go` | mitigate | Share one placement-selection helper between `route-task --dry-run` and persisted routing so preview and commit cannot disagree on owner or target |
| T-08-02-02 | D | `internal/session/up.go`, `internal/runtime/daemon.go` | mitigate | Restrict pane creation, ready-file pane maps, adapter creation, and unread watchers to pane-backed local agents and fail early when none exist |
| T-08-02-03 | I | `internal/session/run_rebuild.go` | mitigate | Render target name, kind, capabilities, and reason from durable `ChildTask.Placement` data instead of mutable config or tmux state |
| T-08-02-04 | R | `internal/runtime/daemon.go` | accept | Full remote notification transport is explicitly deferred; non-pane-backed targets are skipped deterministically rather than pretending local delivery exists |
</threat_model>
