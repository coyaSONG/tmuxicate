<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Derived summary -> CLI output | The summary block becomes operator-visible through `run show` and `task done`, so divergent formatting paths would create untrustworthy output. |
| Task completion metadata -> completion print hook | `newTaskDoneCmd` decides whether to auto-print a run summary based on message metadata from the durable envelope. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | S | `cmd/tmuxicate/main.go` | mitigate | Auto-print the completion summary only when the completed message matches the canonical `root_message_id` from durable envelope metadata for that run. |
| T-05-02-02 | T | `internal/session/run_rebuild.go` | mitigate | Reuse `BuildRunSummary` and `FormatRunSummary` inside `FormatRunGraph` instead of hand-assembling a second CLI-only summary block. |
| T-05-02-03 | R | `cmd/tmuxicate/main.go` | mitigate | Fail loudly if the post-completion `LoadRunGraph` rebuild fails, rather than printing partial output that operators cannot trace back to durable artifacts. |
</threat_model>
