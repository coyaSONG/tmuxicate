<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `RunGraph` -> summary projection | Derived status and ownership leave the validated rebuild layer and become operator-visible summary rows. |
| Source task -> descendant workflow fold-back | Review tasks and rerouted current tasks are folded into one source-task summary row and can be misattributed if linkage rules drift. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | T | `internal/session/run_summary.go` | mitigate | Use only the existing validated `RunGraph` as input and refuse any design that introduces a second filesystem scan or cached summary artifact. |
| T-05-01-02 | S | `internal/session/run_summary.go` | mitigate | Exclude review-task rows and blocker `CurrentTaskID` rows from the logical summary set so descendant tasks cannot spoof standalone logical work items. |
| T-05-01-03 | R | `internal/session/run_summary.go` | mitigate | Require every summary item to carry source task/message references and fold review/blocker refs back onto the source row so operators can trace summary claims to durable artifacts. |
</threat_model>
