# Phase 10 Plan 02: Remote Dispatch Path

## Objective

Dispatch non-pane target work through the new command transport while preserving canonical task artifacts.

## Tasks

1. Invoke dispatch after non-local task creation.
2. Export a stable environment contract for remote launchers.
3. Persist success and failure results without losing the underlying task.

## Verification

- Non-pane routed tasks create dispatch records.
- Configured dispatch commands run with task and target identifiers.
- Dispatch failure degrades target state instead of corrupting run artifacts.
